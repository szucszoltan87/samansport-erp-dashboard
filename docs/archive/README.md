# docs/archive

This directory stores historical planning and design documents that are no longer active but are preserved for reference.

## Why documents end up here

- Plans written for a stack that has since been replaced (e.g. Streamlit-era plans)
- Roadmaps that predate the current product direction
- Implementation plans that have been superseded by newer work

## How to read these documents

These documents reflect thinking and decisions made at a specific point in time.
They are not authoritative for current development.

For the current authoritative guidance, refer to:

- `CLAUDE.md` — development behavior and conventions
- `docs/HARNESS_PLAN.md` — workflow and planning structure
- `docs/ARCHITECTURE.md` — current system architecture
- `docs/PRODUCT_BRIEF.md` — product direction and goals
